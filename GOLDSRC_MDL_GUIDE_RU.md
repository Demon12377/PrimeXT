## 2. Совместимость флагов из Model Viewer с CS 1.6 (GoldSrc)

Пользовательский вопрос: *Что из опций на скриншоте поддерживается в CS 1.6 и как их активировать?*

| Опция в UI | Поддержка в CS 1.6 | Способ активации (в QC или именем) | Описание |
| :--- | :---: | :--- | :--- |
| **Chrome** | ✅ Да | Имя текстуры содержит `chrome` | Эффект отражения окружения. |
| **Additive** | ✅ Да | `$texrendermode "tex" additive` | Эффект свечения (сложение цветов). |
| **Transparent** | ✅ Да | `$texrendermode "tex" masked` | Прозрачность по маске (индекс 255). Текстура должна начинаться с `{`. |
| **Fullbright** | ✅ Да | Имя текстуры содержит `bright` | Текстура всегда ярко освещена. |
| **Flat Shade** | ✅ Да | (По умолчанию для Chrome) | Отключает сглаживание нормалей меша. |
| **Smooth TBN** | ❌ Нет | (Только PrimeXT) | Расчет векторов для аппаратного Normal Mapping. |
| **Two Sided** | ❌ Нет | (Только PrimeXT) | Отрисовка обеих сторон полигона. |
| **Alpha Solid** | ❌ Нет | (Только PrimeXT) | Прозрачность с физической коллизией по маске. |

## 3. Как реализовать "Карту нормалей" в CS 1.6?

В оригинальном движке GoldSrc (CS 1.6) **аппаратной поддержки карт нормалей нет**.

### ❗ Важное уточнение по Detail Textures:
Команда `r_detailtextures 1` в оригинальном CS 1.6 работает **только для текстур мира (карт)**. Она **не применяется** к моделям оружия (`.mdl`). Использование детальных текстур на моделях — это эксклюзивная фича расширенных движков (PrimeXT, Xash3D, Sven Co-op).

### Единственные рабочие методы для CS 1.6:

#### 1. Запекание (Baking) — "Золотой стандарт"
Это единственный способ получить честный объем на FAMAS в CS 1.6.
*   **Суть:** Вы переносите детализацию с высокополигональной модели на текстуру FAMAS в виде теней и бликов.
*   **Что запекать:** Ambient Occlusion (мягкие тени в щелях) и Cavity Map (подсветка острых граней).
*   **Результат:** Модель выглядит объемной и детальной при любом освещении, так как объем "нарисован" на текстуре.

#### 2. Имитация Specular через Chrome
Для металлических частей (затвор, ствол):
*   Выделите эти детали в отдельный меш и назначьте текстуру с приставкой `chrome`.
*   Сделайте текстуру очень темной, почти черной, с мелким серым шумом.
*   В игре это создаст эффект "бегающих" бликов при поворотах, что визуально напоминает работу нормал-маппинга.

## 4. Технические лимиты (Hard Limits) для CS 1.6

Для 100% стабильности в CS 1.6 придерживайтесь этих цифр:
*   **Кости:** 128 (абсолютный предел).
*   **Вершины:** 2048 (рекомендуется), до 8192 (некоторые сборки).
*   **Текстуры:** До 512x512. Для 1024x1024 используйте компилятор с поддержкой `$noclampsize`.
*   **Полигоны:** До 3000-5000 на модель оружия (для сохранения высокого FPS).
